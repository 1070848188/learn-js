    原型及原型链

    # 原型及原型链

    # 原型

    提到原型，大家自然而然会想到：prototype , __proto__ , constructor, 那么它们都代表什么，它们之间又有什么关系？

    那什么是原型呢？你可以这样理解：每一个JavaScript对象(null除外)在创建的时候就会与之关联另一个对象，这个对象就是我们所说的原型，每一个对象都会从原型"继承"属性。

    # prototype

    # 表现

    我们看这个例子

    function Person() {

}
Person.prototype.name = 'panda';
const person1 = new Person()
const person2 = new Person()
console.log(person1.name) // panda
console.log(person2.name) // panda

    当我们创建Person函数后，在控制台去看看这个函数都有哪些属性：

    可以看出Person函数有prototype属性，这个属性就是Person函数的原型对象，所有由Person函数实例化的对象， 都会"继承"Person原型上的属性，所以person1和person2都可以获取到name属性

    # 拥有prototype的函数

    所有函数都有prototype吗？ 其实并不是的，我们这里把函数分为几种

    • 普通函数 (实名或匿名函数)
    • 箭头函数
    • 构造函数/类
    • 生成器函数
    • ...

    有些函数是不具备prototype的

    /**
 * @description
 *  不具备prototype的函数
 *    + 箭头函数
 *    + 基于ES6给对象某个成员赋值函数值的快捷操作
 */
let obj = {
    fn1: function () {
        // 具备 prototype
    },
    fn2() {
        // 不具备 prototype
    }
}

class Fn {
    fn () { } // 这样也不具备
}

    由此可以得出，除了个别函数，其他函数都拥有原型对象，且函数的prototype指向原型对象

    # __proto__

    既然每个函数都可以通过prototype找到原型对象，那么实例化之后的实例对象是否可以找到原型对象呢？

    跟prototype不一样的是，js中每个对象都有__proto__属性，这个属性指向对象的原型对象

    function Person() {

}
var person = new Person();
console.log(person.__proto__ === Person.prototype); // true

    于是我们得到新的关系图

    # constructor

    函数可以通过prototype找到原型对象，实例对象可以通过__proto__属性找到原型对象，那么反过来， 原型对象怎么去找函数和实例对象呢？

    原型对象上给我们提供了一个constructor属性，指向构造函数

    function Person() {

}

var person = new Person();

console.log(person._\\_proto\_\_ == Person.prototype) // true
console.log(Person.prototype.constructor == Person) // true
// 顺便学习一个ES5的方法,可以获得对象的原型
console.log(Object.getPrototypeOf(person) === Person.prototype) // true

    但是一个函数可以生成无数个实例对象，所以原型对象无法去获取实例对象

    由此可以得出，函数可以通过prototype获取原型对象，实例化对象可以通过__proto__获取到原型对象， 同时原型对象又可以通过constructor获取到函数，形成了最终的关系图

    # 原型链

    # 实例与原型

    这时我们再回去看第一个例子，person1和person2是怎么拿到name属性的，它们本身是不存在name的， 因为遵循原型链的查找机制：

    /**
 * 原型链访问
 *  私有属性中是否存在 => 默认基于__proto__找到所属类prototype属性
 *          => 如果所属类中也没有 => 基于类.prototype.__proto__网上找
 *              => 直到Object类  => 最后到null
 *  
 */

    每当获取属性的时候，都会先从私有属性中查找，若没有找到，就会找到对象的__proto__(对象的原型)上,继续同上的查找， 对象私有属性 -> 对象.__proto__一直找到Object，最后找到null，这就是原型链查找机制


    其实原型对象就是通过 Object 构造函数生成的，结合之前所讲，实例的 __proto__ 指向构造函数的 prototype ，所以我们再更新下关系图：

    # Object的原型

    那 Object.prototype 的原型呢？

    null，我们可以打印：

    console.log(Object.prototype.__proto__ === null) // true

    所以 Object.prototype.proto 的值为 null 跟 Object.prototype 没有原型，其实表达了一个意思。

    所以查找属性的时候查到 Object.prototype 就可以停止查找了。
